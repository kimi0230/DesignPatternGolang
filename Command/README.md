### Command 命令模式:
> 將訊息封裝成物件, 以便能用各種不同訊息, 暫佇, 紀律, 復原等方式加以參數化處理

將一個請求封裝為一個物件, 讓你可以用不同的請求對客戶進行參數化; 對請求排隊或紀錄請求日誌, 以及支援可取消的操作

不要為程式碼加上基於猜測的, 實際上不需要的功能. 如果不清楚一個系統是否需要命令模式, 就不要急著實現它.
事實上, 在需要的時候透過重構實現這個模式並不難, 只有在真正需要如取消/恢復等功能時, 
把原本的程式碼重構為命令模式才有意義
